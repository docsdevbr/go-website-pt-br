A linguagem de programação Go é um projeto de código aberto para tornar as pessoas programadoras mais produtivas.

Go é expressivo, conciso, limpo e eficiente. Seus mecanismos de concorrência facilitam a escrita de programas que aproveitam ao máximo máquinas multicore e em rede, enquanto seu sistema de tipos inovador permite a construção flexível e modular de programas. Go compila rapidamente para código de máquina, mas tem a conveniência da coleta de lixo e o poder da reflexão em tempo de execução. É uma linguagem compilada, tipada estaticamente e rápida, que se comporta como uma linguagem tipada interpretada dinamicamente.

Tutorial: Acessando um banco de dados relacional

Acessando bancos de dados relacionais

Uma visão geral dos recursos de acesso a dados do Go.

Abrindo um handle de banco de dados

Você usa o handle de banco de dados Go para executar operações de banco de dados. Após abrir um handle com propriedades de conexão de banco de dados, o handle representa um pool de conexões que ele gerencia em seu nome.

Executando instruções SQL que não retornam dados

Para operações SQL que podem alterar o banco de dados, incluindo INSERT, UPDATE e DELETE, você usa os métodos Exec.

Consultando dados

Para instruções SELECT que retornam dados de uma consulta, use o método Query ou QueryRow.

Usando instruções preparadas

Definir uma instrução preparada para uso repetido pode ajudar seu código a rodar um pouco mais rápido, evitando a sobrecarga de recriar a instrução cada vez que seu código executa a operação no banco de dados.

Executando transações

O objeto sql.Tx expõe métodos que representam a semântica específica da transação, incluindo Commit e Rollback, bem como métodos que você usa para executar operações comuns de banco de dados.

Cancelando operações de banco de dados em andamento

Usando context.Context, você pode fazer com que as chamadas de função e os serviços da sua aplicação parem de executar antecipadamente e retornem um erro quando o processamento não for mais necessário.

Gerenciando conexões

Para alguns programas avançados, pode ser necessário ajustar os parâmetros do pool de conexões ou trabalhar com conexões explicitamente.

Evitando o risco de injeção de SQL

Você pode evitar o risco de injeção de SQL fornecendo valores de parâmetros SQL como argumentos de função do pacote sql.

Desenvolvendo uma atualização da versão principal

Uma atualização da versão principal pode ser muito disruptiva para as pessoas usuárias do seu módulo, pois inclui alterações que quebram a compatibilidade e representa um novo módulo. Saiba mais neste tópico.

Publicando um módulo

Quando você deseja disponibilizar um módulo para outras pessoas desenvolvedoras, você o publica para que ele fique visível para as ferramentas do Go. Após a publicação do módulo, as pessoas desenvolvedoras que importarem seus pacotes poderão resolver uma dependência do módulo executando comandos como go get.

Numeração da versão do módulo

A pessoa desenvolvedora de um módulo usa cada parte do número da versão do módulo para sinalizar a estabilidade e a compatibilidade com versões anteriores. Para cada nova versão, o número da versão de lançamento de um módulo reflete especificamente a natureza das alterações do módulo desde a versão anterior.

Código que cresce com elegância

Um dos principais objetivos de design de Go é a adaptabilidade do código; que seja fácil pegar um design simples e expandi-lo de forma limpa e natural. Nesta palestra, Andrew Gerrand descreve um servidor simples de "bate-papo aleatório" que combina pares de conexões TCP de entrada e, em seguida, utiliza os mecanismos de concorrência, as interfaces e a biblioteca padrão do Go para estendê-lo com uma interface web e outros recursos. Embora a função do programa mude drasticamente, a flexibilidade do Go preserva o design original à medida que ele cresce.
