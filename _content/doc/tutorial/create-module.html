Esta é a primeira parte de um tutorial que apresenta alguns recursos fundamentais da linguagem Go. Se você está apenas começando a usar Go, não deixe de conferir o Tutorial: Introdução ao Go, que apresenta o comando go, os módulos Go e um código Go muito simples.

Neste tutorial, você criará dois módulos. O primeiro é uma biblioteca que deve ser importada por outras bibliotecas ou aplicações. O segundo é uma aplicação chamadora que usará o primeiro.

A sequência deste tutorial inclui sete tópicos breves, cada um ilustrando uma parte diferente da linguagem.

  1. Crie um módulo -- Escreva um pequeno módulo com funções que você pode chamar de outro módulo.
  2. Chame seu código de outro módulo -- Importe e use seu novo módulo.
  3. Retorne e trate um erro -- Adicione um tratamento de erro simples.
  4. Retorne uma saudação aleatória -- Manipule dados em slices (arrays de tamanho dinâmico do Go).
  5. Retorne saudações para várias pessoas -- Armazene pares chave/valor em um mapa.
  6. Adicione um teste -- Use os recursos de teste unitário integrados do Go para testar seu código.
  7. Compile e instale a aplicação -- Compile e instale seu código localmente.
Observação: Para outros tutoriais, consulte Tutoriais.

Pré-requisitos

  • Alguma experiência em programação. O código aqui é bem simples, mas ajuda saber algo sobre funções, laços e arrays.
  • Uma ferramenta para editar seu código. Qualquer editor de texto que você tiver funcionará bem. A maioria dos editores de texto tem bom suporte para Go. Os mais populares são VSCode (gratuito), GoLand (pago) e Vim (gratuito).
  • Um terminal de comando. O Go funciona bem usando qualquer terminal no Linux e Mac, e no PowerShell ou cmd no Windows.

Inicie um módulo que outras pessoas possam usar

Comece criando um módulo Go. Em um módulo, você coleta um ou mais pacotes relacionados para um conjunto discreto e útil de funções. Por exemplo, você pode criar um módulo com pacotes que tenham funções para fazer análises financeiras, para que outras pessoas desenvolvedoras de aplicações financeiros possam usar seu trabalho. Para mais informações sobre o desenvolvimento de módulos, consulte Desenvolvendo e publicando módulos.

O código Go é agrupado em pacotes, e os pacotes são agrupados em módulos. Seu módulo especifica as dependências necessárias para executar seu código, incluindo a versão do Go e o conjunto de outros módulos que ele requer.

À medida que você adiciona ou aprimora funcionalidades em seu módulo, você publica novas versões do módulo. As pessoas desenvolvedoras que escrevem código que chama funções em seu módulo podem importar os pacotes atualizados do módulo e testar com a nova versão antes de colocá-la em produção.

  1. Abra um prompt de comando e acesse seu diretório inicial com cd.

    No Linux ou Mac: 

        cd

    No Windows: 

        cd %HOMEPATH%
  2. Crie um diretório greetings para o código-fonte do seu módulo Go.

    Por exemplo, no seu diretório inicial, use os seguintes comandos: 

        mkdir greetings
    cd greetings
  3. Inicie seu módulo usando o comando go mod init .

    Execute o comando go mod init, informando o caminho do seu módulo -- aqui, use example.com/greetings. Se você publicar um módulo, este deve ser um caminho de onde seu módulo possa ser baixado pelas ferramentas do Go. Esse seria o repositório do seu código.

    Para saber mais sobre como nomear seu módulo com um caminho de módulo, consulte Gerenciando dependências. 

        $ go mod init example.com/greetings
    go: creating new go.mod: module example.com/greetings

    O comando go mod init cria um arquivo go.mod para rastrear as dependências do seu código. Até o momento, o arquivo inclui apenas o nome do seu módulo e a versão do Go que o seu código suporta. Mas, à medida que você adiciona dependências, o arquivo go.mod listará as versões das quais o seu código depende. Isso mantém as compilações reproduzíveis e lhe dá controle direto sobre quais versões do módulo usar.

  4. No seu editor de texto, crie um arquivo para escrever seu código e chame-o de `greetings.go`.
  5. Cole o código a seguir no seu arquivo `greetings.go` e salve o arquivo. 
        package greetings

    import "fmt"

    // Hello retorna uma saudação para a pessoa nomeada.
    func Hello(name string) string {
        // Retorna uma saudação que incorpora o nome em uma mensagem.
        message := fmt.Sprintf("Olá, %v. Bem-vinda!", name)
        return message
    }

    Este é o primeiro código do seu módulo. Ele retorna uma saudação para qualquer código chamador que solicite uma. Você escreverá o código que chama esta função na próxima etapa.

    Neste código, você:

    • Declara um pacote greetings para coletar funções relacionadas.
    • Implementa uma função Hello para retornar a saudação.

      Esta função recebe um parâmetro name cujo tipo é string. A função também retorna uma string. Em Go, uma função cujo nome começa com letra maiúscula pode ser chamada por uma função que não esteja no mesmo pacote. Isso é conhecido em Go como um nome exportado. Para mais informações sobre nomes exportados, consulte Nomes exportados no tour em Go.

    • Declara uma variável message para armazenar sua saudação.

      Em Go, o operador := é um atalho para declarar e inicializar uma variável em uma linha (Go usa o valor à direita para determinar o tipo da variável). Seguindo o caminho mais longo, você poderia ter escrito isso como: 

              var message string
        message = fmt.Sprintf("Olá, %v. Bem-vinda!", name)
    • Use a função Sprintf do pacote fmt para criar uma mensagem de saudação. O primeiro argumento é uma string de formato, e Sprintf substitui o valor do parâmetro name pelo verbo de formato %v. Inserir o valor do parâmetro name completa o texto de saudação.
    • Retorna o texto de saudação formatado para o código chamador.

Na próxima etapa, você chamará essa função de outro módulo.
