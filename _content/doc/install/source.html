Este tópico descreve como compilar e executar o Go a partir do código-fonte. Para instalar com um instalador, consulte Baixe e instale.

Introdução

Go é um projeto de código aberto, distribuído sob uma licença no estilo BSD. Este documento explica como verificar o código-fonte, compilá-lo em sua própria máquina e executá-lo.

A maioria das pessoas usuárias não precisa fazer isso e, em vez disso, instalará a partir de pacotes binários pré-compilados, conforme descrito em Baixe e instale, um processo muito mais simples. Se você quiser ajudar a desenvolver o conteúdo desses pacotes pré-compilados, continue lendo.

Existem duas toolchains oficiais para o compilador Go. Este documento se concentra no compilador e nas ferramentas Go gc. Para obter informações sobre como trabalhar com o gccgo, um compilador mais tradicional que utiliza o backend do GCC, consulte Configurando e usando o gccgo.

Os compiladores Go suportam os seguintes conjuntos de instruções:

amd64, 386
O conjunto de instruções x86, de 64 e 32 bits.
arm64, arm
O conjunto de instruções ARM, de 64 bits (AArch64) e 32 bits.
loong64
O conjunto de instruções LoongArch de 64 bits.
mips64, mips64le, mips, mipsle
O conjunto de instruções MIPS, big-endian e little-endian, de 64 e 32 bits.
ppc64, ppc64le
O conjunto de instruções PowerPC de 64 bits, big-endian e little-endian.
riscv64
O conjunto de instruções RISC-V de 64 bits.
s390x
A z/Architecture da IBM.
wasm
WebAssembly.

Os compiladores podem ter como alvo os sistemas operacionais AIX, Android, DragonFly BSD, FreeBSD, Illumos, Linux, macOS/iOS (Darwin), NetBSD, OpenBSD, Plan 9, Solaris e Windows (embora nem todos os sistemas operacionais suportem todas as arquiteturas).

Uma lista de ports considerados de "primeira classe" está disponível na página wiki ports de primeira classe.

O conjunto completo de combinações suportadas está listado na discussão sobre variáveis de ambiente abaixo.

Consulte a página MinimumRequirements da Wiki do Go para os requisitos gerais do sistema.

Instale binários do compilador Go para o bootstrap

A toolchain Go é escrita em Go. Para compilá-la, você precisa de um compilador Go instalado. Os scripts que fazem a compilação inicial das ferramentas procuram um comando "go" em $PATH, então, desde que você tenha o Go instalado em seu sistema e configurado em $PATH, você pode compilar o Go a partir do código-fonte. Ou, se preferir, você pode definir $GOROOT_BOOTSTRAP como a raiz de uma instalação Go para usar na compilação da nova toolchain Go; $GOROOT_BOOTSTRAP/bin/go deve ser o comando go a ser usado.

A versão mínima do Go necessária depende da versão alvo do Go:

  • Go <= 1.4: uma toolchain C.
  • 1.5 <= Go <= 1.19: um compilador Go 1.4.
  • 1.20 <= Go <= 1.21: um compilador Go 1.17.
  • 1.22 <= Go <= 1.23: um compilador Go 1.20.
  • Daqui para frente, a versão 1.N do Go exigirá um compilador Go 1.M, onde M é N-2 arredondado para baixo para um número par. Exemplo: Go 1.24 e 1.25 exigem Go 1.22.

Existem quatro maneiras possíveis de obter uma toolchain de bootstrap:

  • Baixar uma versão binária recente do Go.
  • Realizar a compilação cruzada da toolchain usando um sistema com uma instalação funcional do Go.
  • Usar o gccgo.
  • Compilar uma toolchain a partir do Go 1.4, a última versão do Go com um compilador escrito em C.

Essas abordagens são detalhadas abaixo.

Toolchain de bootstrap a partir de uma versão binária

Para usar uma versão binária como uma toolchain de bootstrap, consulte a página de downloads ou use qualquer outra distribuição Go empacotada que atenda aos requisitos mínimos de versão.

Toolchain de bootstrap a partir do código-fonte com compilação cruzada

Para realizar a compilação cruzada da toolchain de bootstrap a partir do código-fonte, o que é necessário em sistemas que o Go 1.4 não tinha como alvo (por exemplo, linux/ppc64le), instale o Go em um sistema diferente e execute bootstrap.bash.

Quando executado como (por exemplo): 

$ GOOS=linux GOARCH=ppc64 ./bootstrap.bash

bootstrap.bash realiza a compilação cruzada da toolchain para a combinação GOOS/GOARCH, deixando a árvore resultante em ../../go-${GOOS}-${GOARCH}-bootstrap. Essa árvore pode ser copiada para uma máquina do tipo alvo fornecido e usada como GOROOT_BOOTSTRAP para inicializar uma compilação local.

Toolchain de bootstrap usando gccgo

Para usar o gccgo como toolchain de bootstrap, você precisa fazer com que $GOROOT_BOOTSTRAP/bin/go seja a ferramenta go que vem como parte do gccgo 5. Por exemplo, no Ubuntu Vivid: 

$ sudo apt-get install gccgo-5
$ sudo update-alternatives --set go /usr/bin/go-5
$ GOROOT_BOOTSTRAP=/usr ./make.bash

Toolchain de bootstrap a partir do código-fonte C

Para compilar uma toolchain de bootstrap a partir do código-fonte C, use o branch git release-branch.go1.4 ou go1.4-bootstrap-20171003.tar.gz , que contém o código-fonte do Go 1.4, além das correções acumuladas para manter as ferramentas funcionando em sistemas operacionais mais recentes. (Go 1.4 foi a última distribuição em que a toolchain foi escrita em C.) Após descompactar o código-fonte do Go 1.4, execute cd para o subdiretório src, defina CGO_ENABLED=0 no ambiente e execute make.bash (ou, no Windows, make.bat).

Após descompactar o código-fonte do Go 1.4 no seu diretório GOROOT_BOOTSTRAP, você deve manter o checkout desta instância do git clone no branch release-branch.go1.4. Especificamente, não tente reutilizar este git clone na etapa posterior chamada "Fazer o fetch do repositório". A toolchain de bootstrap do go1.4 deve ser capaz de percorrer corretamente o código-fonte do go1.4 que ela presume estarem presentes nesta raiz do repositório.

Note que o Go 1.4 não roda em todos os sistemas que as versões posteriores do Go rodam. Em particular, o Go 1.4 não oferece suporte às versões atuais do macOS. Nesses sistemas, a toolchain de bootstrap deve ser obtida usando um dos outros métodos.

Instale o Git, se necessário

Para executar a próxima etapa, você precisa ter o Git instalado. (Verifique se você possui um comando git antes de prosseguir.)

Se você não tiver uma instalação do Git funcionando, siga as instruções na página Downloads do Git.

(Opcional) Instale um compilador C

Para compilar uma instalação Go com suporte a cgo, que permite que programas Go importem bibliotecas C, um compilador C como gcc ou clang deve ser instalado primeiro. Faça isso usando qualquer método de instalação padrão no sistema.

Para compilar sem o cgo, defina a variável de ambiente CGO_ENABLED=0 antes de executar all.bash ou make.bash.

Faça o fetch do repositório

Mude para o diretório onde pretende instalar o Go e certifique-se de que o diretório goroot não existe. Em seguida, clone o repositório e faça o checkout da última tag ou branch de lançamento (go1.22.0 ou release-branch.go1.22, por exemplo): 

$ git clone https://go.googlesource.com/go goroot
$ cd goroot
$ git checkout <tag>

Onde <tag> é a string da versão de lançamento.

O Go será instalado no diretório onde for feito o checkout. Por exemplo, se o checkout do Go for feito em $HOME/goroot, os executáveis serão instalados em $HOME/goroot/bin. O diretório pode ter qualquer nome, mas note que se o checkout do Go for feito em $HOME/go, ele entrará em conflito com o local padrão de $GOPATH. Consulte GOPATH abaixo.

Lembrete: Se você optou por compilar também os binários de bootstrap a partir do código-fonte (em uma seção anterior), você ainda precisa executar git clone novamente neste ponto (para fazer o checkout da <tag> mais recente), pois você precisa manter seu repositório go1.4 separado.

(Opcional) Alterne para o branch master

Se você pretende modificar o código-fonte do Go e contribuir com suas alterações para o projeto, mova seu repositório da tag release para o branch master (desenvolvimento). Caso contrário, pule esta etapa. 

$ git checkout master

Instale o Go

Para compilar a distribuição Go, execute: 

$ cd src
$ ./all.bash

(Para compilar no Windows, use all.bat.)

Se tudo correr bem, o processo será concluído com uma saída como: 

ALL TESTS PASSED

---
Installed Go for linux/amd64 in /home/you/go.
Installed commands in /home/you/go/bin.
*** You need to add /home/you/go/bin to your $PATH. ***

onde os detalhes nas últimas linhas refletem o sistema operacional, a arquitetura e o diretório raiz usados durante a instalação.

Para mais informações sobre como controlar a compilação, consulte a discussão sobre variáveis de ambiente abaixo. all.bash (ou all.bat) executa testes importantes para o Go, o que pode levar mais tempo do que simplesmente compilar o Go. Se você não quiser executar o conjunto de testes, use make.bash (ou make.bat).

Testando sua instalação

Verifique se o Go está instalado corretamente compilando um programa simples.

Crie um arquivo chamado hello.go e insira o seguinte programa nele: 

package main

import "fmt"

func main() {
	fmt.Printf("olá, mundo\n")
}

Em seguida, execute-o com a ferramenta go: 

$ go run hello.go
olá, mundo

Se você vir a mensagem "olá, mundo", o Go está instalado corretamente.

Configure seu ambiente de trabalho

Você está quase terminando. Você só precisa configurar um pouco mais.

Como escrever código em Go Aprenda a configurar e usar as ferramentas Go

O documento Como escrever código em Go fornece instruções essenciais de configuração para usar as ferramentas Go.

Instale as ferramentas adicionais

O código-fonte de várias ferramentas Go (incluindo gopls) é mantido no repositório golang.org/x/tools. Para instalar uma das ferramentas (gopls, neste caso): 

$ go install golang.org/x/tools/gopls@latest

Recursos da comunidade

Os recursos habituais da comunidade listados na página de ajuda têm pessoas desenvolvedoras ativas que podem ajudar você com problemas na sua instalação ou no seu trabalho de desenvolvimento. Para aquelas pessoas que desejam se manter atualizadas, há outra lista de e-mails, golang-checkins, que recebe uma mensagem resumindo cada check-in no repositório Go.

Bugs podem ser reportados usando o rastreador de issues do Go.

Acompanhando os lançamentos

Novos lançamentos são anunciados na lista de e-mails golang-announce. Cada anúncio menciona a tag de lançamento mais recente, por exemplo, go1.9.

Para atualizar uma árvore existente para a versão mais recente, você pode executar: 

$ cd go/src
$ git fetch
$ git checkout <tag>
$ ./all.bash

Onde <tag> é a string de versão do lançamento.

Variáveis de ambiente opcionais

O ambiente de compilação do Go pode ser personalizado por variáveis de ambiente. Nenhuma delas é necessária para a compilação, mas você pode definir algumas para substituir os padrões.

  • $GOROOT

    A raiz da árvore Go, geralmente $HOME/go1.X. Seu valor é incorporado à árvore quando ela é compilada e assume como padrão o pai do diretório onde all.bash foi executado. Não há necessidade de definir isso, a menos que você queira alternar entre várias cópias locais do repositório.

  • $GOROOT_FINAL

    O valor assumido pelos binários e scripts instalados quando $GOROOT não é definida explicitamente. O valor padrão é $GOROOT. Se você quiser construir a árvore Go em um local mas movê-la para outro local após a construção, defina $GOROOT_FINAL como o local final.

  • $GOPATH

    O diretório onde normalmente é feito o checkout de projetos Go fora da distribuição Go. Por exemplo, o checkout de golang.org/x/tools pode ser feito em $GOPATH/src/golang.org/x/tools. Executáveis fora da distribuição Go são instalados em $GOPATH/bin (ou $GOBIN, se definida). Os módulos são baixados e armazenados em cache em $GOPATH/pkg/mod.

    O local padrão de $GOPATH é $HOME/go, e geralmente não é necessário definir GOPATH explicitamente. No entanto, se você tiver feito o checkout da distribuição Go em $HOME/go, você deve definir GOPATH como outro local para evitar conflitos.

  • $GOBIN

    O diretório onde executáveis fora da distribuição Go são instalados usando o comando go. Por exemplo, go install golang.org/x/tools/gopls@latest baixa, compila e instala $GOBIN/gopls. Por padrão, $GOBIN é $GOPATH/bin (ou $HOME/go/bin, se GOPATH não estiver definida). Após a instalação, você deverá adicionar este diretório à sua variável $PATH para poder usar as ferramentas instaladas.

    Note que os executáveis da distribuição Go são instalados em $GOROOT/bin (para executáveis invocados por pessoas) ou $GOTOOLDIR (para executáveis invocados pelo comando go; o padrão é $GOROOT/pkg/$GOOS_$GOARCH) em vez de $GOBIN.

  • $GOOS e $GOARCH

    O nome do sistema operacional e arquitetura de compilação alvos. Estas variáveis têm como valores padrão $GOHOSTOS e $GOHOSTARCH, respectivamente (descritos abaixo).

    As opções para $GOOS são android, darwin, dragonfly, freebsd, illumos, ios, js, linux, netbsd, openbsd, plan9, solaris, wasip1 e windows.

    As opções para $GOARCH são amd64 (x86 de 64 bits, o port mais maduro), 386 (x86 de 32 bits), arm (ARM de 32 bits), arm64 (ARM de 64 bits), ppc64le (PowerPC de 64 bits, little-endian), ppc64 (PowerPC de 64 bits, big-endian), mips64le (MIPS de 64 bits, little-endian), mips64 (MIPS de 64 bits, big-endian), mipsle (MIPS de 32 bits, little-endian), mips (MIPS de 32 bits, big-endian), s390x (IBM System z de 64 bits, big-endian) e wasm (WebAssembly de 32 bits).

  • $GOHOSTOS e $GOHOSTARCH.

    O nome do sistema operacional e da arquitetura de compilação do host. O padrão é o sistema operacional e a arquitetura do sistema local.

    As opções válidas são as mesmas de $GOOS e $GOARCH, listadas acima. Os valores especificados devem ser compatíveis com o sistema local. Por exemplo, você não deve definir $GOHOSTARCH como arm em um sistema x86.

  • $GO386 (somente para 386, o padrão é sse2).

    Esta variável controla como o gc implementa cálculos de ponto flutuante.

    • GO386=softfloat: usa operações de ponto flutuante de software; deve suportar todos os chips x86 (Pentium MMX ou posterior).
    • GO386=sse2: usa SSE2 para operações de ponto flutuante; tem melhor desempenho, mas está disponível apenas em Pentium 4/Opteron/Athlon 64 ou posterior.
  • $GOARM (somente para arm; o padrão é detectado automaticamente se estiver compilando no processador alvo, 7, caso contrário).

    Isso define a versão da arquitetura do coprocessador de ponto flutuante ARM que o runtime deve ter como alvo. Se você estiver compilando no sistema alvo, seu valor será detectado automaticamente.

    • GOARM=5: usa ponto flutuante de software; quando a CPU não tiver coprocessador VFP.
    • GOARM=6: usa apenas VFPv1; padrão se estiver compilando de forma cruzada; geralmente núcleos ARM11 ou superiores (VFPv2 ou superior também são suportados).
    • GOARM=7: usa VFPv3; geralmente núcleos Cortex-A.

    Em caso de dúvida, deixe esta variável indefinida e ajuste-a, se necessário, ao executar o executável Go pela primeira vez. A página GoARM na wiki da comunidade Go contém mais detalhes sobre o suporte ARM do Go.

  • $GOAMD64 (somente para amd64; o padrão é v1).

    Isso define o nível de microarquitetura para o qual compilar. Os valores válidos são v1 (padrão), v2, v3, v4. Consulte a página MinimumRequirements da wiki do Go para obter mais informações.

  • $GOMIPS (apenas para mips e mipsle)
    $GOMIPS64 (apenas para mips64 e mips64le).

    Estas variáveis definem se as instruções de ponto flutuante devem ser usadas. Defina como "hardfloat" para usar instruções de ponto flutuante; este é o padrão. Defina como "softfloat" para usar ponto flutuante suave.

  • $GOPPC64 (apenas para ppc64 e ppc64le).

    Esta variável define o nível do processador (ou seja, a versão da Arquitetura do Conjunto de Instruções) para o qual o compilador será direcionado. O padrão é power8.

    • GOPPC64=power8: gera instruções ISA v2.07.
    • GOPPC64=power9: gera instruções ISA v3.00.
  • $GORISCV64 (somente para riscv64).

    Esta variável define o profile da aplicação RISC-V em modo de usuário para o qual compilar. O padrão é rva20u64.

    • GORISCV64=rva20u64: usa apenas extensões RISC-V obrigatórias no profile RVA20U64.
    • GORISCV64=rva22u64: usa apenas extensões RISC-V obrigatórias no profile RVA22U64.
    • GORISCV64=rva23u64: usa apenas extensões RISC-V obrigatórios no profile RVA23U64.
  • $GOWASM (somente para wasm).

    Esta variável é uma lista separada por vírgulas de recursos experimentais do WebAssembly que o binário compilado do WebAssembly tem permissão para usar. O padrão é não usar recursos experimentais.

    • GOWASM=satconv: gera conversões saturadas (sem captura) de float para int
    • GOWASM=signext: gera operadores de extensão de sinal

Note que $GOARCH e $GOOS identificam o ambiente alvo, não o ambiente em que você está executando. Na prática, você está sempre realizando uma compilação cruzada. Por arquitetura, queremos dizer o tipo de binários que o ambiente alvo pode executar: um sistema x86-64 executando um sistema operacional somente de 32 bits deve definir GOARCH como 386, não amd64.

Se você optar por substituir os padrões, defina essas variáveis no seu perfil de shell ($HOME/.bashrc, $HOME/.profile ou equivalente). As configurações podem ser algo como isto: 

export GOARCH=amd64
export GOOS=linux

No entanto, para reiterar, nenhuma dessas variáveis precisa ser definida para compilar, instalar e desenvolver a árvore Go.
